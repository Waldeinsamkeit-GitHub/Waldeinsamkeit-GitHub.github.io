二、核心原因分析

根本问题：Windows 存在两套独立的 SSH 客户端

  1. Windows 系统自带 OpenSSH

    • 路径：C:\Windows\System32\OpenSSH\
    • 手动验证 ssh -T 用的是这个客户端，已成功加载密钥。

  2. Git for Windows 自带内置 SSH

    • Hexo 部署时，**hexo-deployer-git 会调用 Git 工具**，Git 默认使用自己的内置 SSH 客户端；
    • SSH 密钥若存放在自定义目录下，Git 内置 SSH 会无法识别该路径；
    • Git Bash 手动执行 ssh-add 可临时指定密钥，CMD 无此操作，导致部署失败。
    • Git Bash 与 CMD 是独立环境，手动启动的 ssh-agent 不跨环境共享。

Git Bash 仍需手动 ssh-add 的原因

Git Bash 使用独立的 ssh-agent，不会自动继承 Windows 系统服务中的密钥。

每次重启 Git Bash，agent 都会重置，因此必须重新加载密钥。

5.1 什么是 .bashrc

.bashrc 是 Git Bash 的启动自动执行脚本。

每次打开 Git Bash 时，系统都会自动运行该文件内的所有命令。

相当于 Git Bash 的 “开机自启动项”。
