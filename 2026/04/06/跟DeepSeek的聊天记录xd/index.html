1. 方案选择：我需要买服务器和域名吗？

方案类型 代表服务 费用 适合人群
零成本博客平台 WordPress.com, Blogger, Medium, Notion 完全免费 新手、只想专注写作
纯技术路线（首选） GitHub Pages + Hexo/Hugo/VuePress 完全免费 喜欢折腾的开发者
低成本入门方案 阿里云/腾讯云轻量服务器 ￥38/年起 需要更高自由度（如 WordPress）
域名购买 Namecheap, Cloudflare, 阿里云 ￥55/年起 需要专业品牌形象

结论：对于个人技术博客，方案二（静态博客生成器 + 免费托管） 是最佳选择：零成本、高可控、速度快。

2. 方案二详解：静态博客生成器 + 免费托管

2.1 知识储备清单

  • 必备：Git（add、commit、push）、Markdown（10分钟学会）
  • 环境工具：Node.js（Hexo/VuePress）、Go（Hugo）、Ruby（Jekyll）
  • 进阶：HTML/CSS（主题定制）、命令行基础、YAML/TOML（配置文件）

2.2 四大静态博客生成器对比

特性 Hexo Hugo VuePress Jekyll
核心语言 Node.js Go Node.js (Vue) Ruby
构建速度 中等 极快 快速 中等
上手难度 低 中等 中等（需Vue基础） 中等（Ruby环境）
生态/主题 非常丰富 丰富 中等，主打文档 非常丰富
推荐场景 个人博客 大型/多语言博客 技术文档 个人博客（GitHub原生）

选择建议：

  • 前端/JS开发者 → Hexo
  • 追求极致性能、内容量大 → Hugo
  • 熟悉Vue、主要写文档 → VuePress
  • 希望零配置、原生GitHub支持 → Jekyll

2.3 三大免费托管平台对比

特性 GitHub Pages Cloudflare Pages Vercel
流量限制 无硬性限制 较宽松 免费版有带宽限制
国内访问速度 尚可 较慢（可能回源海外） 一般
部署预览 不支持 支持 优秀
核心优势 与GitHub生态无缝集成 全球最快CDN，Workers集成 现代前端框架支持极佳

推荐：GitHub Pages（最省心、零成本、中文教程丰富）。

2.4 推荐组合：Hexo + GitHub Pages

理由：生态成熟、主题/插件海量、教程最多、国内访问友好。

4.5 原理简述：为什么不用移动密钥文件？

SSH 客户端默认从 ~/.ssh/ 目录读取私钥。无论你的 Hexo 项目在哪个文件夹，git push 时都会自动调用该目录下的密钥。因此不需要将密钥移到 Hexo 目录下，也绝不应该把私钥放在项目文件夹中（会有泄露风险）。

4.6 端口被屏蔽的解决方案（443 端口）

如果 ssh -T git@github.com 超时或连接拒绝，可能是防火墙屏蔽了 22 端口。解决方案：让 SSH 走 443 端口。

5. 常见问题排查

问题现象 可能原因 解决方案
hexo deploy 提示 Permission denied SSH 密钥未正确加载或 GitHub 公钥不匹配 参照第 4 节重新配置 SSH
ssh -T 提示 Connection timed out 22 端口被防火墙屏蔽 配置 ~/.ssh/config 走 443 端口（见 4.6）
部署成功但访问 username.github.io 显示 404 仓库名不是 用户名.github.io 删除仓库重新创建，严格按命名规则
修改主题后样式不生效 浏览器缓存 按 Ctrl+F5 强制刷新；检查 _config.yml 主题名是否正确
hexo deploy 要求输入用户名/密码 使用了 HTTPS 而非 SSH 地址 将 _config.yml 中的 repo 改为 SSH 格式
国内访问 GitHub Pages 慢 GitHub 服务器在海外 接入 Cloudflare CDN（见 3.10）或使用 Gitee Pages（有审核）

6. 总结与进阶建议

恭喜！你已经拥有了一个完全免费、属于自己的技术博客。接下来你可以：

  • 自定义域名：在 GitHub Pages 设置中绑定自己的域名。
  • 自动化部署：使用 GitHub Actions 实现 git push 即自动部署。
  • SEO 优化：添加 hexo-generator-sitemap 插件，提交站点地图到搜索引擎。
  • 图床：使用 GitHub 仓库、七牛云或腾讯云 COS 存储图片。
  • 更多主题：探索 Hexo 官方主题站 hexo.io/themes。

遇到任何问题，欢迎随时查阅本指南或搜索社区解决方案。祝你写作愉快！
