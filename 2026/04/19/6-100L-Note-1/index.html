Lecture 1: Introduction

一、计算机与计算思维

1. 1 陈述性知识 vs. 命令性知识

  • 陈述性知识（Declarative Knowledge）：陈述事实（例如： = y y y = x）。计算机无法直接处理此类描述。
  • 命令性知识（Imperative Knowledge）：操作方法/食谱 (Recipe)。编程的本质就是给计算机写食谱。

1. 2 什么是算法（Algorithm）？

算法是一套包含以下要素的指令序列：

  1. 步骤序列：1, 2, 3…
  2. 控制流：判断（如果… 否则…）与循环（重复…）。
  3. 终止条件：何时停止（例如：结果足够接近）。

核心思想：计算机是笨拙但极快的机器。它只擅长两件事：存储海量数据和极快地执行算术运算。它只会严格按照你给的指令执行，不会自行做出判断。

1. 3 编程语言发展简史

  • 固定程序计算机：操作无法存储复用（如计算器）。
  • 存储程序计算机：指令作为数据存储，由解释器执行。
  • 图灵机理论：任何可计算的问题在一种语言中能实现，在另一种语言中也能实现。Python 提供了更友好的原语（Primitives） 来避免直接操作底层硬件。

1. 4 字符串的不可变性 (Immutability)

  • 定义: 字符串一旦创建，无法在原处修改。
  • 后果: 任何试图“修改”字符串的操作，实际上都是创建了一个新的字符串对象。

Python 使用缩进来定义代码块，具体的缩进规则是怎样的？

简单总结：

  • 同一层级的语句，缩进字符数必须完全相同，至于缩进几个空格对逻辑没有影响。如果第一个语句缩进了 4 个空格，第二个缩进了 3 个空格，Python 解释器就会抛出 IndentationError。
  • 不同层级之间，外层用 2 个空格，内层用 100 个空格，逻辑上完全不影响运行结果，只是可读性极差。只要各自层级内部保持一致即可。
  • Python 3 严格禁止混用制表符（Tab）和空格。即使肉眼看起来对齐了，解释器也会报错（TabError）。
  • Python 官方风格指南 PEP 8 强烈建议使用 4 个空格作为一级缩进。

详见 Python 文档/词法分析/缩进

Python 严格缩进的话，该怎么将一个逻辑行拆成多个物理行以便于阅读？

简单总结：

  • 逻辑行由一条或多条 物理行 根据 显式 或 隐式 的 行连接 规则构造而成。

  • 显式拼接行：

    • 两个及两个以上的物理行可用反斜杠（\）拼接为一个逻辑行，规则如下：以不在字符串或注释内的反斜杠结尾时，物理行将与下一行拼接成一个逻辑行，并删除反斜杠及其后的换行符。

    • 以反斜杠结尾的行不能包含注释。反斜杠不能用于延续注释内容。除字符串字面量之外，反斜杠不能用于延续标记（即非字符串字面量的标记不能通过反斜杠拆分到物理行的下一行）。在字符串字面量之外的行中，反斜杠出现在其他位置是非法的。

  • 隐式拼接行：

    • 圆括号、方括号、花括号内的表达式可以分成多个物理行，不必使用反斜杠。

    • 隐式行拼接可含注释；后续行的缩进并不重要；还支持空的后续行。

三、for 循环

3.1 动机：简化固定次数循环

while 循环需要手动管理初始化、条件判断和变量更新，繁琐且易错。for 循环针对遍历序列的场景提供了更简洁的语法。

类比：

  • while：Netflix 连续播放，直到你手动停止或剧集结束。
  • for：Netflix 自动播放完指定数量的剧集（如 4 集）后暂停询问。

3.2 语法

  • variable：循环变量，每次迭代时自动被赋值为序列中的下一个元素。
  • sequence：任何可迭代对象（如 range 生成的数列、字符串、列表等）。

注意：循环变量variable

在 for 循环内部修改循环变量（如 i = 5）不会影响下一次迭代的赋值，Python 仍会按序列顺序赋予新值。

3.3 range() 函数

range() 用于生成整数序列，是 for 循环最常用的搭档。

形式 含义 生成的序列（举例）
range(stop) 从 0 开始，到 stop-1 结束，步长 1 range(5) → 0, 1, 2, 3, 4
range(start, stop) 从 start 开始，到 stop-1 结束，步长 1 range(3, 5) → 3, 4
range(start, stop, step) 从 start 开始，按 step 递增/递减，不包含 stop range(4, 0, -1) → 4, 3, 2, 1

注意：range 的参数含义与字符串切片 [start:stop:step] 逻辑一致，只是分隔符不同（逗号 vs 冒号）。

五、浮点数误差与二进制简介

5.1 一个反直觉的现象

  • 将 0.1 累加 10 次，数学上应得 1.0，但计算机却得到 0.999...，相等比较为 False。
  • 直接计算 0.1 * 10 却能得到 1.0。

5.2 原因：浮点数的二进制表示

  • 计算机内部以二进制（0 和 1）存储所有数据。
  • 整数可以精确转换为二进制（如 1507₁₀ = 10111100011₂）。
  • 但许多十进制小数（如 0.1）在二进制中是无限循环小数，类似于十进制中的 1/3 = 0.333...。
  • 计算机只能用有限位数近似存储这些小数，从而引入微小的舍入误差（Floating-point error）。
  • 反复进行浮点运算时，误差会累积，导致不可预期的结果。

5.3 编程启示

  • 避免直接比较两个浮点数是否相等（==）。
  • 替代方案：检查两数之差是否小于一个极小的阈值（如 abs(a - b) < 1e-9）。
  • 对于需要精确计算的场景（如金融），应使用 decimal 模块或整数（以最小单位计）。

5.4 十进制转二进制的算法思路

整数部分（以 1507 为例）：

  1. 将数除以 2，记录余数（0 或 1），该余数为二进制的最低位。
  2. 将商继续除以 2，记录新的余数，依次作为更高的位。
  3. 重复直到商为 0。

负数只需在结果前添加负号。

Lecture 5: Floats and Approximation Methods

一、浮点数误差的动机回顾

上一讲中我们看到了一个令人困惑的现象：

将 0.1 累加 10 次，在数学上应精确等于 1.0，但计算机却给出了 0.999...，相等比较返回 False。这揭示了一个重要问题：浮点数在计算机中的表示并非总是精确的，直接使用 == 比较浮点数可能导致程序行为异常。

本讲将深入探讨这一现象背后的原因——二进制表示法，并引出第二种重要算法：近似方法（Approximation Method）。

二、十进制小数如何转换为二进制

2.1 基本思路

十进制整数转换为二进制有一套清晰的算法（上一讲已介绍）。但对于小数部分，转换方式则不同。核心思路是：

找到一个足够大的 2 的幂次，使得该小数乘以该幂次后得到一个整数。然后将该整数转换为二进制，再相应地移动小数点。

2.2 示例：将 0.375 转换为二进制

  1. 寻找合适的 2 的幂次：0.375 = 3/8，乘以 8（即 2³）得到整数 3。
  2. 将整数部分转换为二进制：3 的二进制为 11。
  3. 根据幂次移动小数点：乘以 2³ 后，结果需除以 2³ 以恢复原值。在二进制中，除以 2³ 等价于将小数点向左移动 3 位。11 向左移 3 位（不足补 0）得 0.011。

因此，0.375 的二进制表示为 0.011。

2.3 通用转换算法

  • 对于某些小数（如 0.5、0.625），很快就能找到合适的 p，二进制表示较短。
  • 但对于 0.1，程序需要将 p 增大到约 50 才能找到整数乘积，得到的二进制表示长达 50 多位。

2.4 关键问题

并非所有十进制小数都能找到有限长度的二进制精确表示。正如十进制中 1/3 = 0.333... 无限循环，二进制中许多有限小数也会变成无限循环小数。由于计算机存储空间有限，只能截断或舍入，误差由此产生。

三、浮点数在计算机中的存储方式

3.1 科学计数法类比

计算机内部以类似科学计数法的形式存储浮点数，由两部分组成：

  • 有效数字（Significand）：一个整数（实际上以二进制形式存储）。
  • 指数（Exponent）：2 的幂次，指示小数点的位置。

例如，内部表示为 (125, -2) 的含义是：

  • 有效数字 125 转换为二进制。
  • 指数 -2 表示将小数点向左移动 2 位。
  • 最终表示的十进制数为 31.25。

由于小数点位置可随指数“浮动”，这种表示法被称为浮点数（Floating Point Number）。

3.2 精度限制与舍入误差

  • 现代计算机通常使用 32 位或 64 位 来存储浮点数，其中分配给有效数字的位数是固定的。
  • 若一个数的二进制表示超过了可用位数（例如 0.1 需要 50 多位），则多余的部分会被截断（truncate）。
  • 截断引入的误差约在 2⁻³² 数量级（对于 32 位），看似极小，但在反复运算中会累积放大，导致如开头示例所示的反直觉结果。

拓展 - IEEE754 标准

现代计算机普遍遵循 IEEE 754 标准来表示浮点数。该标准定义了单精度（32位）和双精度（64位）的存储格式，包括符号位、指数位（移码表示）和尾数位（隐含前导 1）。了解这一标准有助于理解 0.1 + 0.2 == 0.3 为何为 False 以及 inf、NaN 等特殊值的产生。

3.3 编程启示

  • 绝对不要用 == 直接比较两个浮点数是否相等。
  • 正确做法：判断两数之差的绝对值是否小于一个极小的阈值（epsilon）。

四、近似方法（Approximation Method）

4.1 猜与检查算法的局限

上一讲的“猜与检查”只能判断整数是否为完全平方数（或立方数），无法给出非完全平方数的近似平方根。近似方法正是为了解决这一局限。

4.2 近似方法的基本思想

  1. 从一个已知太小的初始猜测值开始（如 0）。
  2. 以一个很小的步长（increment） 逐步增加猜测值。
  3. 每次检查当前猜测的平方是否已经足够接近目标值 x。
  4. 若在允许的误差范围内，则停止并返回当前猜测作为近似解。

两个关键参数：

  • epsilon：允许的误差范围（如 0.01），决定何时认为“足够接近”。
  • increment：步长（如 0.0001），决定猜测值的精细程度。

  2. 浮点数比较原则：永远不要使用 ==，应使用 abs(a - b) < epsilon 判断是否足够接近。

  3. 近似方法

    ：

    • 针对无法直接求得精确解的问题，通过逐步试探和误差控制，获得一个“足够好”的近似解。
    • 需要定义两个参数：步长 increment 和允许误差 epsilon。

  4. 避免无限循环：当使用固定步长搜索时，必须警惕“跳过目标区间”的风险，应结合额外退出条件（如值已超过目标值）来保证程序终止。

  5. 权衡意识：减小步长可提高精度但牺牲速度；增大 epsilon 可加快收敛但降低精度。实际编程中需根据需求做出合理选择。

Lecture 6: Bisection Search

一、二分查找的动机

1.1 近似方法的回顾与局限

上一讲中我们使用近似方法（Approximation Method） 来计算平方根：

  • 从 0 开始，以极小步长（如 0.0001）递增猜测值。
  • 当猜测值平方与目标值的差小于 epsilon 时停止。
  • 对于较大数值（如 54321），该算法可能需要数千万次循环，耗时数十秒甚至陷入无限循环（因步长过大跳过目标区间）。
  • 必须添加额外退出条件（guess² <= x）才能避免死循环。

这一局限性促使我们寻找一种更高效的算法——二分查找（Bisection Search）。

2.1 猜书页游戏

  • 有序性：搜索空间必须存在明确的顺序（如页码从小到大、字母表顺序）。
  • 反馈机制：每次猜测后能获知“太低”或“太高”的信息，从而确定丢弃哪一半。
  • 区间收缩：每次迭代将搜索范围缩小为原来的一半。

Lecture 7: Decomposition, Abstraction, Functions

一、分解与抽象：从智能手机说起

1.1 现实世界中的黑盒思维

以智能手机为例：

  • 用户只关心输入与输出的关系：按下按钮→屏幕亮起，滑动屏幕→页面切换。
  • 用户无需了解内部电路、芯片、操作系统如何实现这些功能。
  • 这种隐藏内部实现细节、只暴露必要接口的思想称为抽象（Abstraction）。

1.2 抽象如何促成分解

  • 不同制造商分别开发手机的各个组件（屏幕、电池、摄像头），此即分解（Decomposition）。
  • 组件之间通过标准化的接口协作：某组件需要哪些输入，会输出什么结果。
  • 各制造商无需知道其他组件内部如何工作，只需遵循接口规范即可共同完成一部手机的制造。

1.3 在编程中的应用

  • 抽象：将一段完成特定任务的代码封装为“黑盒”，使用者只需知道给它什么输入、它会返回什么输出。
  • 分解：将大型程序拆分为多个独立、可复用的功能模块，各自开发调试，最后组合起来。

4.2 参数传递与执行流程

调用 is_even(3) 时：

  1. Python 查找名为 is_even 的函数对象。
  2. 将实际参数 3 绑定到形式参数 i。
  3. 执行函数体：return 3 % 2 == 0 → return False。
  4. 函数返回 False，调用处被替换为 False。

4.3 内存视角

  • 函数定义：创建一个函数对象，其名（如 is_even）绑定到该对象（一段可执行代码）。
  • 变量赋值：a = 3 在内存中创建整数对象 3 并绑定到名字 a。
  • 函数调用：a = is_even(3) → 先执行函数获得返回值，再将返回值绑定到 a。

三、环境与作用域

3.1 全局环境与局部环境

  • 全局环境（Global Scope）：主程序运行时所在的环境，定义的变量和函数属于全局。
  • 局部环境（Local Scope）：每次函数调用时创建的新环境，函数执行完毕后即销毁。

3.2 变量查找规则（LEGB 简略版）

当在函数内部访问一个变量时，Python 按以下顺序查找：

  1. Local：当前函数内部定义的变量。
  2. Enclosing：外层嵌套函数的变量（后续课程涉及）。
  3. Global：全局环境中的变量。
  4. Built-in：Python 内置的变量/函数。

3.3 访问全局变量 vs 修改全局变量

  • 读取全局变量：如果函数内部未定义同名局部变量，Python 会自动向上查找到全局变量。

五、本讲核心总结

  1. return vs print：
    • return 将值返回给调用者，函数执行即终止；print 仅输出到控制台。
